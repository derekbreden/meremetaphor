Pitfalls That Are Not Solvable

Constant claims of success when failing

"Aha, I see now!" (does not see at all)

"Great, the test is passing!" (by deleting the assertion)

"I've fixed the issue!" (broke 3 other things)

Pathological focus on green checkmarks

  • Will modify tests to match broken behavior rather than fix the code
  • Removes "problematic" assertions instead of addressing root cause
  • Total disregard for higher level goals when fixated on "test passing"

These behaviors persist despite any amount of correction

Pitfalls That Can Be Mitigated

Every hour = brand new hire

  • The only context they have is what you've written down
  • Imagine what you could expect from someone on their first day

Auto-compaction is harmful

  • 100% of the time it's better to manually write summaries
  • Ask the instance to write a document → edit it yourself → provide to next instance
  • Otherwise: confused newbie diversions down wrong paths

"Best Practices" aren't always best

  • Example: Guard assertions
  • Like pulling teeth, but with sufficient prompting can be overcome
  • Need explicit examples and repeated reinforcement
